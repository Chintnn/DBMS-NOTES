# **Database Triggers **  

## **1Ô∏è‚É£ Introduction to Database Triggers**  
üìå **Definition:**  
A **database trigger** is a database object that is created on the client and stored on the server in the Oracle engine‚Äôs system table.  

üìå **Key Components:**  
- A **named database event** (e.g., an `INSERT`, `UPDATE`, or `DELETE` statement).  
- A **PL/SQL block** that executes when the event occurs.  
- The **occurrence of the event** is bound to table data being changed.  

---

## **2Ô∏è‚É£ Uses of Database Triggers**  
‚úÖ Provides a **highly customizable** database management system.  
‚úÖ Can **permit** DML statements (INSERT, UPDATE, DELETE) only at specific times (e.g., regular business hours).  
‚úÖ Helps **maintain an audit trail** üìù by recording table modifications.  
‚úÖ **Prevents invalid transactions** üö´.  
‚úÖ Enforces **complex security authorizations** üîí.  

---

## **3Ô∏è‚É£ Database Triggers and Cascading Triggers**  
üîÑ **Cascading Triggers:**  
- When a **trigger fires**, an SQL statement inside the trigger‚Äôs PL/SQL code block can fire **the same or another trigger**.  
- **‚ö†Ô∏è Excessive use** of cascading triggers may create **complex interdependencies**, making it difficult to maintain large applications.  

---

## **4Ô∏è‚É£ Database Triggers vs. Procedures**  
| Feature | Database Triggers | Procedures |
|---------|------------------|------------|
| **Execution** | Fired **implicitly** by the Oracle engine when an associated table is modified | Must be **explicitly called** by a user |
| **Parameters** | ‚ùå **Do not** accept parameters | ‚úÖ **Can** accept parameters |

---

## **5Ô∏è‚É£ How to Apply Database Triggers**  
A **trigger** has **three main parts**:  

### **1Ô∏è‚É£ Triggering Event**  
üîπ A **SQL statement** that causes the trigger to fire.  
üîπ Example SQL operations that act as triggering events:  
   - `INSERT`  
   - `UPDATE`  
   - `DELETE`  

### **2Ô∏è‚É£ Trigger Restriction**  
üîπ Specifies a **Boolean condition** that must be **true** for the trigger to fire.  
üîπ Used for **triggers that fire for each row**.  
üîπ Implemented using the `WHEN` clause.  

### **3Ô∏è‚É£ Trigger Action**  
üîπ The **PL/SQL block** that executes when the triggering statement occurs.  
üîπ It may contain:  
   - SQL and PL/SQL statements  
   - PL/SQL constructs  
   - Calls to stored procedures  

---

## **6Ô∏è‚É£ Types of Triggers**  
1Ô∏è‚É£ **Row Trigger** ‚Äì Fires **once for each row** affected by the event.  
2Ô∏è‚É£ **Statement Trigger** ‚Äì Fires **once per statement**, regardless of how many rows are affected.  
3Ô∏è‚É£ **Before Trigger** ‚Äì Executes **before** the event occurs.  
4Ô∏è‚É£ **After Trigger** ‚Äì Executes **after** the event occurs.  
5Ô∏è‚É£ **Combination Triggers:**  
   - **Before Statement Trigger**  
   - **Before Row Trigger**  
   - **After Statement Trigger**  
   - **After Row Trigger**  

---

## **7Ô∏è‚É£ Syntax for Creating a Trigger**  
```sql
CREATE OR REPLACE TRIGGER [schema.] <TriggerName>
{BEFORE | AFTER}
{DELETE | INSERT | UPDATE [OF column_name]}
ON [schema.] <table_name>
[REFERENCING {OLD AS old, NEW AS new}]
[FOR EACH ROW [WHEN (condition)]]
DECLARE
    <variable declarations>
BEGIN
    <PL/SQL subprogram>
EXCEPTION
    <Exception handling>
END;
```

---

## **8Ô∏è‚É£ Keywords and Parameters**  
| Keyword | Description |
|---------|------------|
| **REPLACE** | Recreates the trigger if it already exists. |
| **SCHEMA** | The schema in which the trigger exists. |
| **BEFORE** | Fires the trigger **before** executing the event. |
| **AFTER** | Fires the trigger **after** executing the event. |
| **DELETE** | Fires the trigger **when a DELETE statement** is executed. |
| **INSERT** | Fires the trigger **when an INSERT statement** is executed. |
| **UPDATE** | Fires the trigger **when an UPDATE statement** is executed. |
| **ON** | Specifies the table associated with the trigger. |
| **REFERENCING** | Allows the trigger to refer to the **old and new values** of the row. |
| **FOR EACH ROW** | Designates the trigger as a **row trigger**. |
| **WHEN** | Specifies a condition that must be met for the trigger to execute. |

## **9Ô∏è‚É£ Example ‚Äì MySQL Trigger**  
üìå **Scenario:**  
We have an **`emp`** table, and when a record is **deleted**, we want to store the deleted record in another table **`demp`**, along with the user who performed the deletion and the date of the operation.

---

### **Step 2: Creating the Trigger**
```sql
DELIMITER //
CREATE TRIGGER t1
AFTER DELETE ON emp
FOR EACH ROW
BEGIN
    INSERT INTO demp VALUES (
        OLD.empno, OLD.ename, OLD.job, OLD.deptno, OLD.salary, OLD.comm, DATABASE(), CURDATE()
    );
END;
//
DELIMITER ;
```

## **üîπ Summary ‚Äì Key Points for Exam Revision üîπ**  
‚úÖ **Database triggers** are objects stored in the **Oracle system table** and execute automatically when triggered by events.  
‚úÖ **Uses of triggers**: Customization, audit trails, security enforcement, and transaction validation.  
‚úÖ **Triggers vs. Procedures**: **Triggers are automatic**, **procedures must be called manually**.  
‚úÖ **Components of a trigger**: **Triggering event, restriction, action**.  
‚úÖ **Types of triggers**: **Row, Statement, Before, After, Combination triggers**.  
‚úÖ **Cascading triggers** can cause **complex dependencies** and are **difficult to maintain**.  
‚úÖ **MySQL example**: **Deletes an employee record** and **stores it in an audit table (`demp`)**.
